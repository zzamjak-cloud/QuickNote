# 페이지 및 DB 항목 페이지

### 페이지 컨텐츠 정렬 이슈

*   페이지 내부 컨텐츠가 오른쪽 댓글 공간을 미리 의식해서, 댓글이 없는 상태에서도 해당 빈 공간을 유지하고 있음.
    
    *   "전체 너비" 활성화 상태에서도 댓글 공간이 빈 상태로 유지되는 현상.
        

### 페이지 댓글

*   페이지 상단 "+댓글 추가"시 텍스트 크기가 너무 작다. (가독성 이슈) → 일반 텍스트 크기로 통일시킬 것.
    
*   "작성자 | 내용 | 작성 시간 | 버튼" 컬럼 형식의 레이아웃 변경하고, 모두 한 라인에 표현한다. (컬럼 레이아웃의 통일감 유지)
    
    *   "작성자 | 내용" 은 좌측 정렬
        
    *   "작성 시간 | 버튼"은 우측 정렬
        
    *   각 컬럼 라인 유지하고, 내용은 멀티 라인(줄바꿈) 허용
        
    *   모든 정보는 상단 정렬 처리
        
*   페이지 댓글 수정버튼 클릭시, 기존에 입력했던 내용 없이 완전히 새로 작성하도록 처리되고 있음. → 기존 작성한 텍스트를 필드에 넣어줘야 함. (처음부터 다시 작성하려는 것이 아니라 기존 텍스트를 수정하려는 의도가 더 강하기 때문)
