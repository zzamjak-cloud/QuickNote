# 노션 이관 계획 (Notion ZIP -> QuickNote)

## 0. 목표와 원칙

- 목표: Notion Export ZIP을 QuickNote로 단계적으로 이관한다.
- 원칙:
  - 한 번에 완벽 이관을 목표로 하지 않고, "작동하는 최소 이관(MVP) -> 검증 -> 확장" 순서로 진행한다.
  - 원본 손상 방지를 위해 항상 "미리보기(Preview) -> 실제 반영(Apply)" 2단계로 처리한다.
  - 실패한 항목은 전체 중단 대신 스킵/리포트하고, 재시도 가능한 구조로 만든다.

---

## 1. 범위 정의

### 1-1. 1차(MVP) 범위

- 페이지 계층 구조(폴더/하위 페이지)
- 일반 텍스트 블록
  - heading, paragraph, bullet/numbered list, to-do, quote, code block, divider
- 링크/기본 인라인 스타일(bold/italic/code)
- 이미지/파일 첨부(가능한 범위)
- 데이터베이스
  - CSV 기반 테이블 데이터(row/column)
  - 기본 속성(text, number, select, multi-select, date, checkbox)

### 1-2. 2차 확장 범위

- relation/rollup/formula/created_by/last_edited_by 등 고급 DB 속성
- toggle, callout, synced block, embed 등 고급 블록
- 멘션(user/page/date) 정밀 매핑
- 권한/댓글/히스토리 등 Notion 고유 메타 보존 전략

### 1-3. 비범위(초기)

- 100% 픽셀 단위 렌더 동일성
- Notion 권한 모델 완전 복제
- Notion 자동화/통합(app integration) 이식

---

## 2. 아키텍처 초안

## 2-1. 파이프라인

1. ZIP 업로드
2. 압축 해제 및 manifest 생성
3. 파일 분류(md/csv/assets)
4. Notion 중간 포맷(Normalized JSON) 변환
5. QuickNote 모델 변환
6. Preview 생성
7. 검토 후 Apply(실제 저장)
8. 결과 리포트(성공/실패/스킵)

### 2-2. 핵심 모듈

- `notionZipParser`: ZIP 해제, 파일 인덱싱, Notion export 구조 파악
- `notionMdNormalizer`: Markdown/링크/에셋 경로 정규화
- `notionDbImporter`: CSV -> QuickNote database 변환
- `importPlanner`: 생성/업데이트/스킵 계획 산출
- `importExecutor`: 트랜잭션 단위 반영 + 실패 격리
- `importReporter`: 검증/에러/누락 항목 리포트 생성

### 2-3. 데이터 안전장치

- dry-run 기본값
- idempotent key(재실행 시 중복 생성 방지)
- chunk 처리(대용량 ZIP 보호)
- 실패 시 부분 롤백 또는 "실패 항목만 재시도"

---

## 3. 단계별 구현/테스트 계획

## Phase 1. 분석기(읽기 전용) 구축

### 작업

- ZIP 업로드 및 해제
- 파일 트리/페이지 계층 추출
- md/csv/assets 개수와 용량 통계 생성
- Notion export 패턴 샘플 수집

### 완료 기준

- 어떤 ZIP이 들어와도 "읽기"는 안정적으로 동작
- 최소 통계 리포트(JSON) 생성 가능

### 테스트

- 소형/중형/대형 ZIP 샘플
- 한글 파일명/공백/특수문자 경로
- 깨진 링크/누락된 asset 참조

---

## Phase 2. Markdown -> QuickNote 페이지 MVP

### 작업

- heading/paragraph/list/todo/quote/code/divider 매핑
- 내부 링크 상대경로 해석
- 이미지/파일 경로 매핑(1차)
- 지원 불가 블록은 "fallback block"으로 보존

### 완료 기준

- 대표 문서 20개 기준 본문 손실 없이 생성
- 지원 불가 요소는 누락되지 않고 fallback으로 기록

### 테스트

- 블록 단위 스냅샷 테스트
- 원문/결과 비교(줄수, 블록수, 링크수)
- 렌더링 수동 QA(깨짐/순서/인코딩)

---

## Phase 3. CSV -> QuickNote DB MVP

### 작업

- CSV header -> property 자동 매핑 규칙
- 기본 타입 추론(text/number/select/multi-select/date/checkbox)
- DB row import + invalid row 격리

### 완료 기준

- 샘플 DB 10개에서 컬럼 누락 없이 이관
- 타입 추론 실패 컬럼은 안전하게 text fallback

### 테스트

- 빈 셀/혼합 타입/잘못된 날짜/쉼표 포함 텍스트
- 대용량 row(1만+) 성능 측정
- import 후 조회/정렬/필터 기본 동작 확인

---

## Phase 4. Preview + Apply 워크플로우

### 작업

- Preview 화면: 생성될 페이지/DB/스킵/경고 목록 표시
- 충돌 정책 선택(건너뛰기/이름 변경/덮어쓰기 제한)
- Apply 실행 전 최종 확인

### 완료 기준

- 사용자가 실제 반영 전 변경량을 명확히 확인 가능
- Apply 실패 시 실패 항목만 재시도 가능

### 테스트

- 동일 ZIP 재실행(idempotency)
- 이름 충돌, 경로 충돌, 중복 자산
- 중간 실패(네트워크/스토리지) 복구

---

## Phase 5. 고급 매핑 확장

### 작업

- relation/rollup/formula 보강
- 고급 블록(toggle/callout/embed 등) 확장
- 정밀 링크 복원(페이지 id 기반)

### 완료 기준

- 상위 중요 워크스페이스 이관 정확도 향상
- "fallback 비율" 단계적 감소

### 테스트

- 실제 사용자 ZIP 회귀 테스트 세트 운영
- 버전별(노션 export 변화) 호환성 체크

---

## 4. 매핑 규칙 (초안)

- Notion heading 1/2/3 -> QuickNote heading 1/2/3
- bullet/numbered list -> 동일 타입 list
- to-do -> checkbox block
- code fence -> code block(language 유지 시도)
- table in markdown -> 우선 "단순 표 블록" 또는 text fallback
- CSV DB:
  - 숫자형 판별 실패 시 text
  - date 파싱 실패 시 원문 text 저장
  - select/multi-select 후보값은 import 중 자동 option 등록

---

## 5. 품질 지표 (KPI)

- 페이지 본문 보존율: 95%+
- DB 컬럼 보존율: 99%+
- 링크 복원율: 90%+
- import 실패율: 2% 이하
- 재실행 중복 생성률: 0%

---

## 6. 운영/릴리즈 전략

- 기능 플래그로 제한 공개(`notionImportBeta`)
- 내부 워크스페이스 -> 파일럿 사용자 -> 전체 공개 순서
- 실패 리포트 자동 수집(에러 코드/파일 경로/블록 타입)
- 릴리즈마다 "지원 블록/속성 매트릭스" 갱신

---

## 7. 즉시 실행할 첫 스프린트 (권장)

1. Phase 1 완료 (읽기 전용 분석기 + 통계 리포트)
2. Phase 2 일부 완료 (본문 핵심 블록 6종 우선)
3. Preview JSON 출력까지 연결
4. 실제 저장은 막아두고(dry-run only) 샘플 ZIP 3개 검증
